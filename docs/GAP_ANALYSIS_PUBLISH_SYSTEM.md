# 功能差距分析报告 (Current vs. Unified Publish Design v2.0)

本项目当前的发布系统已实现了**动态表单引擎**的核心框架，但在内容深度和交互广度上与 `v2.0` 设计文档仍存在以下明显差距。

## 一、 分类全景图 (Category Panorama)

| 分类 | 设计文档 v2.0 状态 | 当前实现状态 | 差距说明 |
|:---|:---:|:---:|:---|
| 💰 卖东西 (GOODS) | 已定义 | ✅ 完整实现 | 逻辑最全，支持分角色表单 |
| 🔄 出租/租赁 (RENTAL) | 已定义 | ⚠️ 基础实现 | 目前使用通用模板，缺少“押金”、“取还规则”等关键字段 |
| 🙋 找人帮忙 (TASK) | 已定义 | ⚠️ 基础实现 | 目前使用通用模板，缺少“期望时间”、“报酬方式”等 |
| 💼 提供服务 (SERVICE) | 已定义 | ⚠️ 基础实现 | 缺少“子类型选择”（按时/上门/报价等），目前字段太单一 |
| 🎁 免费赠送 (FREE_GIVEAWAY) | 已定义 | ❌ 未实现 | 入口和配置均不存在 |
| 🔍 求购/求租 (WANTED) | 已定义 | ❌ 未实现 | 入口和配置均不存在 |
| 🎉 社区活动 (EVENT) | 已定义 | ❌ 未实现 | 入口和配置均不存在 |

## 二、 字段矩阵深度 (Field Matrix)

当前大部分分类（除 GOODS 外）仍在使用 `defaultBasicFields` (仅标题/图片/价格/描述)。
- **缺失重点**：
    - **租赁类**：租期单位、配件清单、损坏赔偿条款。
    - **服务类**：预约时段 (Schedule)、取消政策、服务半径 (Radius)。
    - **任务类**：紧急程度标识、技能标签。

## 三、 表单系统功能 (Form System)

1.  **信息完整度评分 (Completeness Score)**:
    *   实现情况：逻辑代码已存在，分数值已计算。
    *   **差距**：设计文档中提到的“高分激励提示”（3倍曝光、优质发布徽章）尚未在 UI 中展示。
2.  **智能提示 (Smart Assistance)**:
    *   **差距**：目前完全没有根据标题进行“类别猜测”或“自动填充分类”。
3.  **模板系统 (Template System)**:
    *   **差距**：文档提到的“一键套用模板”（如搬家模板、相机租赁模板）功能尚不存在。

## 四、 UI/UX 体验

1.  **发布步骤**:
    *   实现：2步流程（选分类 -> 填详情）。
    *   **设计目标**：设计文档建议 4步流程（起草 -> 分类 -> 详情 -> 预览），**预览环节**目前缺失。
2.  **草稿箱**:
    *   实现：前端 `localStorage` 自动保存。
    *   **差距**：缺乏显式的“恢复草稿”或“草稿记录管理”界面。

---

## 下一步建议 (Priority)

1.  **P0 - 补齐字段矩阵**：为 `RENTAL` 和 `SERVICE` 创建独立的配置文件，替换目前的通用模板。
2.  **P1 - 完善分类入口**：在发布页增加“免费赠送”和“求购/求租”入口。
3.  **P1 - 预览功能**：在正式提交前增加预览页面，让用户确认展示效果。
4.  **P2 - 智能分类推荐**：引入简单的标题关键词匹配逻辑。
